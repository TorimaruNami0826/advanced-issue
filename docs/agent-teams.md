# Agent Teams 設計書

## 追加機能の概要
ステータス自動更新：担当者がアサインされたら自動で「対応中」に変わる
集計API：ステータス別の件数を返すAPIを追加する

## Agent構成
| Agent名 | 役割 | 担当タスク |
|---|---|---|
| Backend Agent | バックエンド開発 | 集計APIの新規実装、および担当者アサイン時のステータス自動更新ロジックの実装 |
| Frontend Agent | フロントエンド開発 | 担当者アサイン時に自動更新されたステータスが画面に即時反映される処理の実装 |
| Test Agent | テスト自動化 | 追加された集計APIと自動更新ロジックのユニットテストコードの作成、実行 |
| Review Agent | コード品質監査 | 実装されたコードにバグがないか、バリデーションやセキュリティに問題がないかのレビュー |

## タスクの依存関係
Backend Agent（APIとロジックの実装） 
  → Frontend Agent（画面への反映処理） 
  → Test Agent（追加機能のテスト作成・実行） 
  → Review Agent（全体のコードレビュー・最終確認）

## 並列実行できるタスク
- Backend Agent が集計APIの実装を進めている間に、Frontend Agent は画面側の集計表示用UIの準備（デザインや枠組みの作成）を並列で進めることができる
- Test Agent は、Backend Agent のAPI実装が完了した時点でバックエンドのユニットテストの作成を並列して開始できる

## 統合時の確認ポイント
- 画面から担当者をアサインした際、エラーにならずにデータベースのstatusが「対応中」に変わるか
- 新設した集計APIを叩いた際、正しい件数がJSON形式で返ってくるか
- 追加機能の実装によって、前半で作った既存の基本CRUD機能やテストが壊れていないか
