# Subagent定義書

## backend-architect
- 役割：バックエンドのAPIおよびデータベースの実装・設計担当
- 担当するタスク：テーブル構築、APIエンドポイント（一覧取得・登録・詳細取得・更新・削除・コメント追加）の実装
- 特に注意させること：db_comments.inquiries_noなど、テーブル間の紐付けが正しく動作するように実装すること
                      db_inquiries.inquiries_noやdb_assignees.assignees_idに、データの自動連番を設定すること

## frontend-developer
- 役割：ユーザーが操作する管理画面（UI）の実装担当
- 担当するタスク：問い合わせ内容の登録フォーム、一覧表示、詳細確認ができる画面の実装
                  ステータスや担当者で一覧を絞り込める検索機能の実装
                  バックエンドの各APIと通信し、画面上での登録・更新・削除・コメント投稿が即時反映されるようにする
- 特に注意させること：システム保守担当者が使いやすいスッキリしたレイアウトにすること

## security-auditor
- 役割：システム全体のセキュリティ対策とバリデーションの監査担当
- 担当するタスク：問い合わせ登録やコメント追加時に、不正なスクリプトなどが送信されないよう、入力値のバリデーションやエスケープ処理を徹底すること
                  ステータスに、仕様にない不正な文字列が保存されないようにバックエンド側でチェックすること
- 特に注意させること：実行時にセキュリティ関連の不具合やエラーが出ないこと
